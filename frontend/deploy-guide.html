Project Structure

Your downloaded zip contains two folders. Here's what everything does.

Folder layout
vidai/
├── frontend/
│   └── index.html          # The website — goes on GitHub Pages
│
└── backend/
    ├── app.py              # Python Flask server — goes on Render
    ├── requirements.txt    # Python packages to install
    └── render.yaml         # Render.com config (auto-deploy)
💡
The frontend and backend are deployed separately and talk to each other via API calls. This is how all modern web apps work.

Flow of a video request

1
User opens your site (GitHub Pages)

Types a topic and clicks Generate Video.

2
Frontend sends request to backend (Render)

JavaScript sends the topic, voice, duration to your Python server via API.

3
Backend generates the video

Ollama writes script → Pexels fetches images → Edge TTS makes voice → MoviePy renders MP4.

4
Video uploaded to Cloudflare R2

The finished MP4 is stored in R2 and a public URL is returned to the user.

1
Sign up at render.com

Go to render.com → Sign up with your GitHub account (easier — it connects automatically).

2

3
Configure the service

Fill in these settings exactly:

SettingValue

Root Directorybackend

4. Ollama on Render
Backend · Step 4

AI Model on Render

The free Render tier doesn't have enough RAM to run Ollama (4GB needed). Here are your two options:

☁️
Option A — OpenAI API (Recommended)
Use GPT-3.5-turbo for script writing. $5 free credit on signup. Very fast, no RAM issues.
💻
Option B — Render paid tier ($7/mo)
Upgrade to Render's Starter plan for enough RAM to run Ollama locally on the server.

Option A — Use OpenAI (easiest)

1. Sign up at platform.openai.com → Get API key (free $5 credit)
2. Add OPENAI_API_KEY to your Render environment variables
3. In app.py, replace the write_script() function with:

Replace write_script() in app.py
import openai

def write_script(topic, duration, style):
    word_count = int(duration * 2.2)
    client = openai.OpenAI(api_key=os.environ.get("OPENAI_API_KEY"))
    res = client.chat.completions.create(
        model="gpt-3.5-turbo",
        messages=[{
            "role": "user",
            "content": f"Write a {word_count}-word video narration about: {topic}. Style: {style}. Just the spoken words, no labels."
        }]
    )
    return res.choices[0].message.content.strip()

Also add openai to your requirements.txt, push to GitHub, and Render will auto-redeploy.

✅
With OpenAI's free $5 credit you can generate roughly 500+ videos before paying anything. More than enough to get started.

5. Cloudflare R2
Storage · Step 5

Cloudflare R2 Storage

Generated videos need to be stored somewhere publicly accessible. R2 gives you 10GB free — plenty for hundreds of videos.

1
Sign up at cloudflare.com

Create a free account. No credit card needed for R2's free tier.

2
Create an R2 bucket

Go to R2 Object Storage in the sidebar → Create bucket → Name it vidai-videos → Create.

3
Enable public access

Inside your bucket → Settings → Public Access → Allow public access. Copy the Public Bucket URL — this is your R2_PUBLIC_URL.

4
Get API credentials

Go to R2 → Manage R2 API Tokens → Create API Token → Give it Object Read & Write permission. Copy the Access Key ID and Secret Access Key.

5
Find your endpoint URL

Your R2 endpoint looks like: https://ACCOUNT_ID.r2.cloudflarestorage.com — find it in your R2 bucket settings.

Now go back to Render → Environment and fill in all the R2 variables from Step 3.

💡
R2 free tier includes 10GB storage and 10 million read operations per month. That's roughly 1,000+ videos stored simultaneously with no cost.
